form 태그

      - div와 같은 영역  태그(block 형식)
      - 내부에 작성된 input 태그의 값을 서버로 전달하는 역할
      

      1) action : form 태그 내부에 작성된 값을 전달할 서버 또는 페이지 주소를 작성
        (미작성 시 기본값으로 현재 페이지의 URL 적용)
      2) name : form 태그의 이름을 지정하여 form 태그를 구분하는 용도(JS 사용)    
      3) method(방식) : 서버로 데이터 전달하는 방식을 지정하는 속성값
      

      - GET/ POST 지정 가능(기본값 : GET 방식)
      - get 방식은 url에 내가 요청한 입력값이 그대로 노출되므로 보안 취약(검색, 이동페이지 요청 시 사용)
      - 로그인 회원가입 같은 보안적인 데이터를 get으로 보내면 안됨
      - post 방식은 입력한 값이 url에 노출되지 않으므로 보안 유리(로그인/ 회원가입에 사용 가능)
      4) target : action의 지정된 주소를 현재탭/새탭 중 어디서 열지 지정
